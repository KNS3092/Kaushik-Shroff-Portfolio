MUSI Cache Coherence Protocol

Project Overview: This project involved implementing the MUSI (Modified–Update–Shared–Invalid) cache coherence protocol in the gem5 simulator to support commutative updates, following the principles of the COUP architecture. The protocol extended the baseline MSI implementation using gem5’s SLICC domain-specific language, enabling update-based propagation of shared data across private L1 caches. The design introduced a new Update (U) state and added multiple custom coherence messages to support local accumulation of shared variables without invalidations. Custom microbenchmarks were developed to test race scenarios and validate data convergence across sharers. Protocol rules and transitions were tested under controlled workloads and syntactic correctness was verified through build validation and trace inspection.

Key Takeaways:

  • Implemented MUSI cache coherence protocol in gem5 by extending MSI with support for commutative updates.
  • Defined new states, transitions, and messages in SLICC to support update-based consistency and avoid invalidations.
  • Introduced an Update (U) state to propagate commutative operations across private caches without loss of correctness.
  • Developed custom test workloads to validate protocol correctness under shared-memory conflicts and data races.
  • Verified implementation through build success, trace analysis, and protocol event logging in the Ruby memory model.
