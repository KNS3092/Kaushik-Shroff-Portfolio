O3 CPU Power Model

Project Overview: This project focused on developing a power modeling framework for the Out-of-Order (O3CPU) processor in gem5, addressing the gap between performance simulation and power estimation. By leveraging McPAT and CACTI, activation energy values were extracted for key CPU components, allowing the model to estimate dynamic power consumption based on execution statistics.

To validate the model, various workloads—including DAXPY, SAXPY, and integer-based benchmarks like the Sieve of Eratosthenes and Random Branch programs—were tested. Results confirmed expected power trends, such as integer workloads consuming up to 2.8× more power than floating-point workloads and random branch workloads exhibiting the highest branch predictor power consumption. The model provides valuable insights for optimizing power-performance trade-offs in modern CPU architectures.

Key Takeaways:

  • Developed a power model for gem5’s O3CPU by integrating activation energy values from McPAT and CACTI.
  • Tested power consumption across multiple workloads, confirming expected trends such as higher power use in integer operations (~3.7W) compared to floating-point (~1.5W).
  • Validated results against prior models, with the O3CPU consuming up to 4.1W, aligning with known out-of-order CPU power characteristics.
  • Enabled section-based power analysis to identify energy hotspots in workload execution.
  • Future improvements include better SIMD instruction modeling, real-time power estimation in gem5, and integration of static power consumption metrics.
